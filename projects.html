Mandelbrot Movie

This is a project developed in C (C99 Programming Language Standard) to calculate and visualize the Mandelbrot Set. The main function behind this project is motivated by the Mandelbrot function, M(Z, C) = Z^2 + C. Where Z is the center, C is a constant, and Z is an iteration variable. We are interested in how many times Z can iterate, and we set a maximum iteration value because some functions can iterate indefinitely. M and C can both be imaginary numbers. The program also deals with how colors are represented when plotting the Mandelbrot function on a canvas. We use the P6 color pallete where each unique color corresponds to a unique iteration value that is modulated to a set number. To accelerate the number of for loops in the code, parallel programming was implemented to optimize the computation across the cores of the CPU.

The video is my own fractal zoom calculated at (0.37280535024, -0.2551419600000041) with 1750 maximum iterations.

Exploring MNIST Neural Networks

Animation credits: @msvanjie

With deep interest in artificial intelligence, I was able to explore and experience implementing neural networks in different environments. Using the MNIST handwritten digit database, I implemented MNIST neural networks in assembly with RISC-V ISA, in Python with the use of the TensorFlow library, and in Python without the use of TensorFlow.

Implementing the MNIST neural network in assembly was for a school project in UC Berkeley's CS 61C: Machine Structures This project was special as RISC-V was originated in UC Berkeley. The project is a simple, barebone implementation of the MNIST neural networks using operations such as vector inner product, matrix multiplications, and thresholding. The model consists of input matrix, a hidden layer, and output neurons. Because there are write and read matrix operations in the model, it can be trained and improved as well as taking user inputted digit that is 28*28 pixels. This was my first time programming in assembly and implementing a neural network, so it definitely got my hands dirty.

Implementation of MNIST neural network with the use of TensorFlow library was a personal exploration, just to see how a lower level language compares to higher level implementation of neural networks. Obviously, the use of TensorFlow's helper functions make the code simple and clean. But knowing what goes under the hood in TensorFlow's library was insightful.

Implementing a MNIST neural network in Python without the TensorFlow library was for UC Berkeley's CS 188: Introduction to Artificial Intelligence. Instead, NumPy was used to handle the matrix operations of this neural network. After implementing the same model differently three times, I understand the why libraries such as TensorFlow and PyTorch are popular due to their convenience.

Scheme Interpreter

This is a project developed in Python. This was the final project for UC Berkeley's CS 61A: Structure and Interpretation of Computer Programs. The project runs a REPL environment of Scheme in the Python shell, complete with all Scheme functions and properties. It even throws exceptions when it detects errors and displays the corresponding error type. The project exercises all the fundamentals learned from this course such as object oriented programming, list comprehension, linked lists, recursion, tail recursion, and more. Overall, it was fun to make a programming language run another programming language!

AI Pacman

A series of projects written in Python for UC Berkeley's CS 188: Introduction to Artificial Intelligence. The project implements artificially intelligent properties to the retro game of Pacman.

Search algorithms were used to optimally find the best paths for Pacman to win. Search algorithms implemented were depth first search, breadth first search, uniform cost search, and A* search. Also implemented multiple heuristics to optimally find the most efficient and rewarding path for Pacman in multiple mazes.

Game trees were implemented such as minimax, alpha-beta pruning with minimax, and expectimax game trees. Also implemented evaluation functions to optimally play a classic game of Pacman with ghosts, factoring each variable, such as if power pallets are all eaten or nearby, or if the ghosts are scared.

The third part involves using reinforcement learning with implementations of Markov decision processes, Q-learning, and policies.

the last part was machine learning

RISC-V CPU

A simulation of a single cycle RISC-V CPU created using logic circuits in Logisim. The CPU has 8 registers and has modern architecture and datapath such as pipelining. Simple RISC-V ISA can be loaded in and the simulated CPU can output results. This was the final project of UC Berkeley's CS 61C: Machine Structures.

AI PT (with Google's Teachable Machine)

I started losing weight and getting fit with a personal trainer. Having a personal trainer when exercising definitely helps because he/she motivates you. But after my gym membership ended, I had to exercise by myself. I found that hard to do. So I used Google's Teachable Machine version 2.0 to create my own free personal trainer.

I trained models that correctly determine if a posture for an exercise is correct or incorrect. Programmed with voice feedback, the AI trainer counts and confirms that you are exercising correctly. I've included negative voice feedback for correction or slow pace and positive voice feedback for motivation. It is currently a prototype, and purely a demo to test out Google's machine learning model. You can try the prototype yourself using your webcam!
